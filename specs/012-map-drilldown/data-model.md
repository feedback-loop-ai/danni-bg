# Phase 1 Data Model: SVG choropleth + oblast‚Üímunicipality drill-down

**Feature**: 012-map-drilldown
**Date**: 2026-06-16

This feature introduces **no new persisted runtime store**. It adds bundled static data (the 265-municipality gazetteer, real municipality geometry, and the crosswalk) and extends one derived view model (`RegionSummary`) with the parent oblast. All authoritative Bulgarian names are preserved verbatim (Constitution X); gazetteer‚Üîboundary joins use official codes, never names.

## Municipality gazetteer entry (bundled, source of truth)

`src/enrich/gazetteer/municipalities-bg.json` ‚Äî one entry per Bulgarian municipality (265 total). Generated by `packages/geo-boundaries/scripts/generate-municipalities.ts` from `lau-bg.geojson`; consumed by the curate extractor and `generate-crosswalk.ts`.

| Field | Type | Notes |
|-------|------|-------|
| `id` | string | `geo:bg-municipality-<slug>`; slug de-duplicated so ids are unique (FR-009) |
| `labelBg` | string | Authoritative Cyrillic name, verbatim from LAU `name` (Constitution X) |
| `labelEn` | string | Derived English label (title-cased transliteration of `labelBg`) ‚Äî a helper, not authoritative |
| `oblastId` | string | Parent oblast entity id (`geo:bg-oblast-*`), derived spatially (FR-007) |
| `aliases` | string[] | e.g. `["–û–±—â–∏–Ω–∞ <name>"]` so the curate extractor matches both phrasings |
| `lauId` | string | Official Eurostat LAU id (e.g. `SML10`); the crosswalk join key (FR-008) |

**Rules**: exactly 265 entries; every entry has a non-empty `oblastId` (zero unmatched ‚Äî FR-007/SC-002); every `id` is unique (FR-009/SC-006).

**Example**:
```json
{
  "id": "geo:bg-municipality-dospat",
  "labelBg": "–î–æ—Å–ø–∞—Ç",
  "labelEn": "Dospat",
  "oblastId": "geo:bg-oblast-smolyan",
  "aliases": ["–û–±—â–∏–Ω–∞ –î–æ—Å–ø–∞—Ç"],
  "lauId": "SML10"
}
```

## Boundary feature (bundled GeoJSON)

A standard GeoJSON `Feature` (validated by `boundaryFeatureSchema` in `packages/geo-boundaries/src/schema.ts`). Two collections: `oblasts.geojson` (reused unchanged from 008) and `municipalities.geojson` (regenerated, real LAU geometry).

| Field | Type | Notes |
|-------|------|-------|
| `type` | `"Feature"` | |
| `geometry` | `Polygon` \| `MultiPolygon` | Real LAU 2021 geometry (EPSG:4326); the SVG choropleth projects this |
| `properties.boundaryFeatureId` | string | Municipalities keyed `lau-<LAU_ID>`; the crosswalk FK |
| `properties.level` | `"oblast" \| "municipality"` | |
| `properties.ekatte` | string? | `^[0-9]{5}$` when present |
| `properties.iso3166_2` | string? | `^BG-[0-9]{2}$` when present (oblasts) |

**Source**: `packages/geo-boundaries/data/source/lau-bg.geojson` is the raw Bulgaria-filtered Eurostat input; `municipalities.geojson` is the emitted, keyed collection.

## Crosswalk entry (bundled join table)

`packages/geo-boundaries/data/crosswalk.json` ‚Äî static rows joining a gazetteer `entityId` to a `boundaryFeatureId` by official code. Validated by `crosswalkEntrySchema` (Zod, `.strict()`).

| Field | Type | Notes |
|-------|------|-------|
| `entityId` | string | `^geo:bg-(oblast\|municipality)-[a-z0-9-]+$`; must exist in the gazetteer |
| `level` | `"oblast" \| "municipality"` | |
| `boundaryFeatureId` | string | FK into the matching GeoJSON collection |
| `ekatte` | string \| null | `^[0-9]{5}$`; null for oblasts |
| `lauId` | string \| null | **new in this feature** ‚Äî official LAU id; null for oblasts |
| `iso3166_2` | string \| null | `^BG-[0-9]{2}$`; null for municipalities |

**Validation rules** (`superRefine`):
- **oblast** entries: `ekatte === null`, `lauId === null`, `iso3166_2` required.
- **municipality** entries: `iso3166_2 === null`, and at least one of `ekatte` / `lauId` present (the LAU-derived set carries `lauId`).
- The contract JSON (`contracts/regions-api.md` companion `geo-crosswalk` shape) additionally tracks `oblastEntityId` per municipality (the parent oblast); `knownGaps` lists any gazetteer unit intentionally unmapped (each still reachable via search/national grouping).

**Cross-checks** (CI tests, `packages/geo-boundaries/tests/`): no orphan crosswalk rows; every `boundaryFeatureId` exists in the GeoJSON; every `entityId` exists in the gazetteer.

## RegionSummary (derived view model ‚Äî extended)

The per-unit choropleth aggregate returned by `GET /api/regions` (`apps/explorer-api/src/schemas.ts`, computed by `regions-aggregate.ts`). This feature **adds `oblastEntityId`**.

| Field | Type | Notes |
|-------|------|-------|
| `entityId` | string \| null | `geo:bg-oblast-*` / `geo:bg-municipality-*`; null for unlinked boundary units |
| `level` | `"oblast" \| "municipality"` | drives layer styling + drill-down |
| `labelBg` | string | authoritative name (verbatim) |
| `labelEn` | string \| null | derived label |
| `boundaryFeatureId` | string | join key into the GeoJSON the SVG paints |
| `datasetCount` | integer | de-duplicated in-scope count; for oblasts, the union of direct + municipality-rolled-up links counted once (FR-016) |
| `hasData` | boolean | false ‚Üí reserved "no data" colour bucket, still rendered |
| `maxConfidence` | number | strongest geo-link confidence among contributing datasets |
| `oblastEntityId` | string \| null | **new** ‚Äî parent oblast for a municipality; drives client grouping + drill-down (FR-013); null for oblasts |
| `flagged` | `"unlinked"`? | boundary unit with no gazetteer/crosswalk link |

**Aggregation rules** (`aggregateRegions`):
- Per dataset, links are collapsed to the strongest confidence per target region, so a dataset reaching one region by several links counts once at its strongest placement.
- `rollup(linkEntityId) ‚Üí regionIds[]` maps a municipality link up to its parent oblast for oblast-level counts (FR-016); defaults to identity at municipality level.
- `parentOf(entityId)` resolves `oblastEntityId` from the `part_of` knowledge graph; null when absent.

## Pure client view structures (in-memory, render-time)

These are computed by the pure `lib/` modules and consumed by `MapView.tsx`; none are persisted.

### ProjectedFeature (`lib/projection.ts`)
| Field | Type | Notes |
|-------|------|-------|
| `boundaryFeatureId` | string | links to the RegionSummary for shading |
| `d` | string | SVG path data in viewBox space |
| `cx` / `cy` | number | label anchor (polygon centroid); non-finite ‚Üí 0 |
| `bounds` | `[[x0,y0],[x1,y1]]` | bbox in viewBox space; feeds `fitTransform` for drill-down |

### FitTransform (`lib/projection.ts`)
`{ k, x, y }` ‚Äî scale + translate that fits an oblast's `bounds` into the viewBox (with padding) when drilling in.

### Colour bucket / legend (`lib/map-scale.ts`)
Bucket index `0‚Äì5` for a count given `rampBreakpoints(max)` (bucket 0 = no data); `legendStops(max, isDark)` ‚Üí `{ color, from }[]` for the legend.

## State transitions (client)

- **Click oblast (country view)** ‚Üí set drill-down focus to that oblast's `entityId`; apply `fitTransform(focusedOblastBounds)`; render the focused oblast's municipalities (filtered by `oblastEntityId === focus`); scope the dataset list to the oblast via `selectRegion`. (FR-010)
- **Click "‚Üê –ù–∞–∑–∞–¥ –∫—ä–º –æ–±–ª–∞—Å—Ç–∏—Ç–µ"** ‚Üí clear focus; return to the national oblast layer. (FR-011)
- **Click municipality (drilled in)** ‚Üí toggle-select the municipality; dataset list reflects it. (FR-012)
- **Any region selection** ‚Üí writes into `filters.geoUnitIds` for the dataset list + chat only; the region-layers fetch uses `regionFilters` (selection-independent), so the choropleth does not re-scope. (FR-014/FR-015)

## Constitution alignment notes

- **Locale (X)**: `labelBg` is the verbatim Cyrillic LAU name; `labelEn` is a clearly-derived transliteration. The gazetteer‚Üîboundary join uses `lauId`/EKATTE/ISO codes, never names.
- **Type safety (VII)**: the bundled crosswalk + GeoJSON are Zod-validated at load (`schema.ts`); `lauId` is a non-empty string and codes are regex-checked, failing fast on a malformed bundle.
- **Read-only (I)**: every structure here is either bundled static data or a projection of mirror reads; no write path is introduced.
